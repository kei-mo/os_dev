## メモリに書き込む箇所の逆アセンブル


### `bootpack.c`の`HariMain()`の中の逆アセンブルする該当箇所

```
void HariMain(void)
{
  int i;
  for (i = 0xa0000; i <= 0xaffff; i++){
    _write_mem8(/* int addr= */ i, /* int data = */ 15);  // 第2引数dataは 0-15 の値を期待するものである.
  }

  for(;;){
    _io_hlt();
  }
}
```


### 逆アセンブル結果(`./debug/bootpack.hrb.dasm`)

なお,16バイトアラインメントについては以下の記事参照

- [x86-64において一般的なスタックフレーム構造の解説 - ニコニコ動画](https://www.nicovideo.jp/watch/sm34645847)

- [x86-64 モードのプログラミングではスタックのアライメントに気を付けよう - uchan note](http://uchan.hateblo.jp/entry/2018/02/16/232029)
- [スタックフレームのアライメント要件 - Qiita](https://qiita.com/yohhoy/items/54633faf0546f8cd0e3a)
- [c - Stack allocation, padding, and alignment - Stack Overflow](https://stackoverflow.com/a/1061942/9316234)
  - GCCオプション`-mpreferred-stack-boundary=n`
  - > The default value for n is 4 i.e. it will try to align to 16-byte boundaries
- [アセンブリに触れてみよう - Qiita](https://qiita.com/kaito_tateyama/items/89272098f4b286b64115)
